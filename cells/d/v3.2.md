## 发行版源仓库格式规范

### 仓库结构

```plaintext
lina-distros/
  INDEX             # 机器可读的发行版目录（由 build_INDEX.py 生成）
  distros/
    <name>.sh       # 发行版定义文件（bash 5.3+，通过子命令调用）
    ...
```

### INDEX 格式

INDEX 文件为 JSON 文档，由 `build_INDEX.py` 生成。

```json
{
  "version": "LINA_DISTRO_INDEX v3.2",
  "entries": [
    {
      "name": "alpine",
      "path": "distros/alpine.sh",
      "desc": "Alpine Linux",
      "archs": ["aarch64", "x86_64"],
      "versions": ["latest-stable"],
      "mirrors": [
        "https://mirrors.tuna.tsinghua.edu.cn/alpine/",
        "https://mirrors.aliyun.com/alpine/",
        "https://dl-cdn.alpinelinux.org/alpine/"
      ],
      "hash": "<sha256-hex>"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | string | 格式标识符，固定为 `LINA_DISTRO_INDEX v3.2` |
| `entries` | array | 发行版条目列表 |
| `entries[].name` | string | 发行版名称，由定义文件 basename 去除 `.sh` 后缀得出 |
| `entries[].path` | string | 发行版定义文件在仓库内的相对路径 |
| `entries[].desc` | string | 发行版描述文本 |
| `entries[].archs` | array of strings | 支持的 CPU 架构列表 |
| `entries[].versions` | array of strings | 可用的版本列表 |
| `entries[].mirrors` | array of strings | 镜像站基础 URL 列表 |
| `entries[].hash` | string | 对应 `.sh` 脚本文件内容的 SHA-256 哈希值 |

### 发行版定义文件契约

每个 `distros/<name>.sh` 文件通过**子命令模式**调用，不得有隐式副作用。

**子命令：`info`（无参数）**

输出发行版元数据到 stdout（JSON 格式）：

```json
{
  "desc": "<单行描述>",
  "archs": ["<arch>", ...],
  "versions": ["<version>", ...],
  "mirrors": ["<base mirror URL>", ...]
}
```

- `desc` — 单行描述（string）
- `archs` — 支持的架构列表（string 数组，非空）
- `versions` — 可用的版本列表（string 数组，非空）
- `mirrors` — 镜像站基础 URL 列表（string 数组，非空）

**子命令：`get <version> <arch> [mirror]`**

根据给定版本、架构和可选镜像输出下载信息到 stdout（JSON 格式）：

```json
{
  "src": "<rootfs 下载地址>",
  "hash_val": "<算法:hex字符串>"
}
```

- `src` — rootfs 直接下载 URL（string），由传入的 mirror 基础 URL 拼接
- `hash_val` — 校验值（string），格式为 `"算法:hex字符串"` 或 `"SKIP"`。
  支持的算法：`md5`、`sha1`、`sha256`、`sha384`、`sha512`、`sha3-256`、`sha3-384`、`sha3-512`、`blake2b`、`blake2s`、`b2`。

与 v3.1 的区别：
- **移除 `hash_cmd` 字段** — 算法信息不再作为独立字段输出，而是编码进 `hash_val` 的 `"算法:"` 前缀中
- **`hash_val` 格式变更** — 从纯 hex 字符串变为 `"算法:hex字符串"` 格式

消费者必须容忍未知字段保证兼容性，防止意外解析失败。

**错误处理：**

- 参数缺失或无效时，输出使用说明到 stderr 并以退出码 1 结束
- 网络错误时，输出错误信息到 stderr 并以退出码 1 结束
- 不识别子命令时，输出使用说明到 stderr 并以退出码 1 结束

### 嵌入式 sh 模板

```bash
#!/usr/bin/env bash

# shellcheck shell=bash

set -euo pipefail
shopt -s inherit_errexit 2> /dev/null || true

info() {
    cat << 'EOF'
{
  "desc": "<DESCRIPTION>",
  "archs": ["<ARCH>", "..."],
  "versions": ["<VERSION>", "..."],
  "mirrors": ["<MIRROR_URL>", "..."]
}
EOF
}

get() {
    local default_mirror='<DEFAULT_MIRROR_URL>'
    local version="${1}" arch="${2}" mirror="${3:-$default_mirror}"
    [[ -z $version || -z $arch ]] && usage

    # TODO: fetch and compute src, hash_algo, hash_val
    # hash_val 格式: "算法:hex字符串" 或 "SKIP"

    cat << EOF
{
  "src": "${src}",
  "hash_val": "${algo}:${hash_val}"
}
EOF
}

usage() {
    echo "usage: $0 {info | get <version> <arch> [mirror] }" >&2
    exit 1
}

main() {
    case "${1:-}" in
        info)
            info
            ;;
        get)
            get "${2:-}" "${3:-}" "${4:-}"
            ;;
        *) usage ;;
    esac
}

[[ ${BASH_SOURCE[0]} == "${0}" ]] && main "$@"
```

### 新增发行版

1. 创建 `distros/<name>.sh`，按上述契约实现 `info` 和 `get` 子命令
2. 运行 `python3 build_INDEX.py > INDEX` 重新生成索引
3. 运行 `python3 validate.py` 验证下载链接可用性
