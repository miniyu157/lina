## 发行版源仓库格式规范

### 仓库结构

```plaintext
lina-distros/
  INDEX             # 机器可读的发行版目录（由 build_INDEX.py 生成）
  distros/
    <name>.sh       # 发行版定义文件（bash 5.3+，通过子命令调用）
    ...
  ... 其他存储发行版小程序的文件夹
```

### 发行版定义文件契约

每个 `distros/<name>.sh` 文件通过**子命令模式**调用，不得有隐式副作用。

**子命令：`info`（无参数）**

输出发行版标识信息到 stdout（JSON 格式）：

```json
{
  "name": "<发行版名称>",
  "desc": "<单行描述>"
}
```

- `name` — 发行版名称（string），不含路径和扩展名
- `desc` — 单行描述（string）

**子命令：`options`（无参数）**

输出发行版可用选项到 stdout（JSON 格式）：

```json
{
  "archs": ["<arch>", ...],
  "versions": ["<version>", ...],
  "mirrors": ["<base mirror URL>", ...]
}
```

- `archs` — 支持的架构列表（string 数组，非空）
- `versions` — 可用的版本列表（string 数组，非空）
- `mirrors` — 镜像站基础 URL 列表（string 数组，非空）

**子命令：`get <version> <arch> [mirror]`**

根据给定版本、架构和可选镜像输出下载信息到 stdout（JSON 格式）：

```json
{
  "src": "<rootfs 下载地址>",
  "hash_val": "<算法:hex字符串>"
}
```

- `src` — rootfs 直接下载 URL（string），由传入的 mirror 基础 URL 拼接
- `hash_val` — 校验值（string），格式为 `"算法:hex字符串"` 或 `"SKIP"`。
  支持的算法：`md5`、`sha1`、`sha256`、`sha384`、`sha512`、`sha3-256`、`sha3-384`、`sha3-512`、`blake2b`、`blake2s`、`b2`。

消费者必须容忍未知字段保证兼容性，防止意外解析失败。

**错误处理：**

- 参数缺失或无效时，输出使用说明到 stderr 并以退出码 1 结束
- 网络错误时，输出错误信息到 stderr 并以退出码 1 结束
- 不识别子命令时，输出使用说明到 stderr 并以退出码 1 结束

### 嵌入式 sh 模板

```bash
#!/usr/bin/env bash

# shellcheck shell=bash

set -euo pipefail
shopt -s inherit_errexit 2> /dev/null || true

info() {
    cat << 'EOF'
{
  "name": "<NAME>",
  "desc": "<DESCRIPTION>"
}
EOF
}

options() {
    cat << 'EOF'
{
  "archs": ["<ARCH>", "..."],
  "versions": ["<VERSION>", "..."],
  "mirrors": ["<MIRROR_URL>", "..."]
}
EOF
}

get() {
    local default_mirror='<DEFAULT_MIRROR_URL>'
    local version="${1}" arch="${2}" mirror="${3:-$default_mirror}"
    [[ -z $version || -z $arch ]] && usage

    # TODO: fetch and compute src, hash_val
    # hash_val 格式: "算法:hex字符串" 或 "SKIP"

    cat << EOF
{
  "src": "${src}",
  "hash_val": "${algo}:${hash_val}"
}
EOF
}

usage() {
    echo "usage: $0 {options | info | get <version> <arch> [mirror] }" >&2
    exit 1
}

main() {
    case "${1:-}" in
        options)
            options
            ;;
        info)
            info
            ;;
        get)
            get "${2:-}" "${3:-}" "${4:-}"
            ;;
        *) usage ;;
    esac
}

[[ ${BASH_SOURCE[0]} == "${0}" ]] && main "$@"
```

### INDEX 格式

INDEX 文件为 JSON 文档，由 `build_INDEX.py` 扫描 `distros/` 目录，调用每个脚本的 `info` 和 `options` 子命令后编排生成。

`build_INDEX.py` 对每个脚本做以下加工：
- 将 `info` 和 `options` 的输出合并重组，而非原样透传
- 计算脚本文件的 sha256，取其前 7 位追加到 entry `name` 后缀，构成 `发行版名.xxxxxxx`
- 将 `options` 的输出整体嵌套在 `exec.options` 中

```json
{
  "version": "v3.3",
  "entries": [
    {
      "name": "alpine.bbe502c",
      "desc": "Alpine Linux",
      "exec": {
        "file": "distros/alpine.sh",
        "options": {
          "archs": ["aarch64", "x86_64"],
          "versions": ["latest-stable"],
          "mirrors": [
            "https://mirrors.tuna.tsinghua.edu.cn/alpine/",
            "https://mirrors.aliyun.com/alpine/",
            "https://dl-cdn.alpinelinux.org/alpine/"
          ]
        }
      }
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | string | 格式标识符，固定为 `v3.3` |
| `entries` | array | 发行版条目列表 |
| `entries[].name` | string | `发行版名.文件sha256前7位`，由 `info.name` 拼接文件哈希得出 |
| `entries[].desc` | string | 发行版描述文本，来自 `info.desc` |
| `entries[].exec` | object | 执行信息 |
| `entries[].exec.file` | string | 发行版定义文件在仓库内的相对路径 |
| `entries[].exec.options` | object | 发行版的可用选项，即 `options` 子命令的输出 |
| `entries[].exec.options.archs` | array of strings | 支持的 CPU 架构列表 |
| `entries[].exec.options.versions` | array of strings | 可用的版本列表 |
| `entries[].exec.options.mirrors` | array of strings | 镜像站基础 URL 列表 |

### 与 v3.2 的区别

- **`info` 子命令拆分** — `info` 原来返回全部元数据（name/desc/archs/versions/mirrors），现在只返回 `name` 和 `desc`；架构、版本、镜像信息移至新增的 `options` 子命令
- **INDEX 结构重组** — 移除顶层 `path`、`archs`、`versions`、`mirrors`、`hash` 字段，改为嵌套在 `exec` 对象中（`exec.file`、`exec.options`）；文件哈希不再作为独立字段，而是追加到 entry `name` 后缀（取 sha256 前 7 位）
- **`build_INDEX.py` 使用方式变更** — 不再隐式扫描固定目录，改为接受目录名参数：`python3 build_INDEX.py distros > INDEX`

### 新增发行版

1. 创建 `distros/<name>.sh`，按上述契约实现 `info`、`options` 和 `get` 子命令
2. 运行 `python3 build_INDEX.py distros > INDEX` 重新生成索引
3. 运行 `python3 validate.py` 验证下载链接可用性
